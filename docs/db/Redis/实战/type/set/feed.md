# feed流

关注推送,feed流，直译投喂

通过刷新，推送消息给你

两种模式：

- TimeLine：不做内容筛选，按时间排序，朋友圈、关注动态

  优点：信息全面，不缺失

  缺点：信息噪音多，用户不一定喜欢感兴趣

- 智能排序：通过算法推送用户感兴趣的内容，抖音...

  优点：投喂用户兴趣，用户易粘性沉迷

  缺点：算法不准确则起反作用

---

TimeLine的三种解决方案：

- 推

  类似消息队列，往订阅的用户推送消息

- 拉

  

- 推拉

  对于数据量多的使用拉，数据量少用推，频繁使用用推，基本不用也是拉
